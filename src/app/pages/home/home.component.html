DSP Transformation Quick Start Guide

Elements of a Formula

Formulas in Field Mapping are made of reusable logic elements:

  • Literal Value: Static values like "Hello", 42, or true.
  • Field Reference: Points to data from other fields, blocks, or user input.
  • Global Variable: Predefined system variables like User.ProfileId or User.Email.
  • Function: Logic helpers like IF(), ADD_DAYS(), etc.
  • Operator: Math or logic symbols: +, ==, AND, etc.
  • APEX CLASS: (Advanced) Custom backend logic for developers.

Purpose of Field Mapping

Definition: Field Mapping sets the metadata for target fields and defines the transformation logic DSP uses to calculate and assign their values.

Key Concept: Instead of editing records directly on the page layout, use the Fields Mapper tool to auto-generate mapping records based on your configuration.

Using the Fields Mapper Tool

What It Does: Automatically generates Field Mapping records.

Why Use It: It ensures that your mappings are consistent with the DSP runtime engine requirements, reducing manual errors.

Key Components in Field Mapping

Component Description
Field Mapping API Name Unique identifier for each mapping record.
Mapping (Transformation Logic) Contains the logic used to compute the target field’s value.
Createable & Updateable Flags Indicate if a field can be inserted or modified.
Nillable Specifies if the target field is allowed to have a null value.
External Id Field(s) Used when working with reference fields (lookup or master-detail).
Reference To & Relationship Name Define the object and relationship for reference fields.

Steps to Get Started

  1. Access the Fields Mapper: Use this tool to create or modify mappings without directly editing the page layout.
  2. Define Your Target Fields: Identify the fields that need transformation.
  3. Set Up Transformation Logic: Use the Mapping field to write the logic that computes each target field’s value.
  4. Review Field Attributes: Check properties like Createable, Updateable, Nillable, and External Id configurations to ensure they match your data requirements.
  5. Test Your Configuration: Validate that the DSP runtime engine correctly applies the transformations during processing.
